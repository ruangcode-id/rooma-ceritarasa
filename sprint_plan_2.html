Minggu 2 – 3
A
Dev A
Reservasi Full Flow API
  • Public reservasi API (tanpa login) critical
  • Pilih tanggal + session + jumlah tamu
  • Auto-assign meja logic
  • Generate & validate cancel_token
  • Cancel reservasi via token endpoint
  • Admin filter/update status reservasi API
  • Validasi capacity & blocked dates

Dev B
Payment Integration API
  • Midtrans Snap SDK integration integration
  • Create payment (deposit & full) API
  • Webhook handler Midtrans callback webhook
  • Payment status tracking & update
  • Refund flow logic
  • Admin payment list & filter API
C
Dev C
Check-in + Notification System API
  • Check-in flow API (scan/manual) critical
  • Update status → checked_in / no_show
  • Push notification setup (VAPID keys)
  • Push subscription management API
  • Notification triggers: reservasi, cancel, check-in, payment integration
  • Notification list & read status API

Business Flow Breakdown
📅
Reservasi Flow
Dev A · Public + Admin
  • Guest pilih tanggal & session
  • Validasi capacity & blocked dates
  • Auto-assign meja tersedia
  • Generate cancel_token unik
  • Return reservationId → Dev B
  • Admin dapat update & filter status
💳
Payment Flow
Dev B · Midtrans Integration
  • Terima reservationId dari Dev A
  • Init Midtrans Snap session
  • Guest pilih deposit / full payment
  • Webhook callback dari Midtrans
  • Update payment status di DB
  • Trigger notifikasi → Dev C
✅
Check-in & Notif
Dev C · Real-time System
  • Subscribe push notifications (VAPID)
  • Listen events: reservasi, payment, cancel
  • Admin scan QR / input manual
  • Update status → checked_in / no_show
  • Kirim notif ke admin & guest
  • Notif list & read status tracking
